# Сумма прописью v2.3

Статический сервис для расчёта НДС и подготовки готовых формулировок суммы прописью для договоров, актов, счетов и других документов.

Домен: `summa.martsynkovskyi.ru`

## Возможности

- расчёт НДС по ставкам 22% и 5%;
- режимы «НДС включён», «НДС сверху» и «Без НДС»;
- четыре формата записи суммы прописью;
- готовые формулировки «в том числе НДС» и «кроме того НДС»;
- варианты обоснования для операций без НДС;
- копирование отдельного результата или всего расчёта;
- автоматическое склонение рублей и копеек;
- корректный ввод сумм с пробелами, точкой или запятой;
- локальное сохранение текущего черновика;
- история десяти последних расчётов с восстановлением, копированием и удалением;
- работа без интернета после первого открытия;
- адаптивный интерфейс для телефонов, планшетов и десктопов;
- локальный шрифт Golos Text без CDN и внешних запросов;
- автоматические тесты расчётов на странице `tests.html`.

## Горячая клавиша

`Ctrl + Enter` в Windows/Linux или `Cmd + Enter` в macOS копирует весь расчёт одним блоком.

## Публикация на GitHub Pages

1. Загрузить содержимое проекта в корень репозитория.
2. В `Settings → Pages` выбрать публикацию из нужной ветки и корневой папки.
3. Убедиться, что файл `CNAME` содержит:

```text
summa.martsynkovskyi.ru
```

4. После публикации открыть сайт один раз онлайн. Service Worker сохранит основные файлы, после чего сервис сможет работать без интернета.

## Проверка расчётов

Открыть:

```text
https://summa.martsynkovskyi.ru/tests.html
```

Страница проверяет:

- разбор российских и смешанных числовых форматов;
- расчёт НДС 22% и 5%;
- режимы «включён», «сверху» и «без НДС»;
- склонения рублей и копеек;
- четыре формата суммы прописью;
- граничное значение суммы.

## Безопасность и приватность

- backend отсутствует;
- внешние библиотеки, CDN, аналитика и трекеры не используются;
- расчёты выполняются в браузере;
- черновик и история сохраняются только в `localStorage` текущего браузера;
- пользовательские значения выводятся через `textContent`, без пользовательского `innerHTML`;
- применяется Content Security Policy без `unsafe-inline`;
- Service Worker кэширует только собственные статические файлы сайта.

## Шрифт

# Обновление до версии 2.4

1. Удалите старые файлы проекта из репозитория, сохранив историю Git.
2. Загрузите содержимое этого архива в корень репозитория.
3. Убедитесь, что `index.html`, `manifest.webmanifest`, `service-worker.js` и `CNAME` лежат в корне.
4. Дождитесь завершения GitHub Pages deployment.

Сборка использует версионированные CSS и JavaScript, новый cache name и стабильный PWA ID `/`. Старый кэш не должен смешиваться с новой версией.
